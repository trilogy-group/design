# Product Context

## Purpose

This single-page site explains design context for coding agents. It teaches how focused design skills, the Impeccable skill, and Google's DESIGN.md format work together.

## Audience

The audience is developers, designers, and AI-coding practitioners who already understand agents but need a short, concrete map for using design context without turning every UI into a generic demo page.

## Tone

Precise, tactile, and technical. The page should feel like a design-system workbench: useful, opinionated, and visually memorable.

## Anti-References

Avoid generic SaaS hero sections, purple gradients, glass panels, repeated icon-card grids, fake metrics, and vague claims. Use concrete labels and short explanations.

## Strategic Principles

- Show the relationship between judgment and format: Impeccable critiques and guides craft; DESIGN.md stores design facts and rationale.
- Keep every content block short enough to present verbally.
- Use real visual structure, readable type, strong contrast, and direct copy.
